class Gyroscope : public codal::CodalComponent#

Class definition for Gyroscope.

Public Functions

Gyroscope(CoordinateSpace &coordinateSpace, uint16_t id = DEVICE_ID_GYROSCOPE)#

Constructor. Create a software abstraction of an gyroscope.

Parameters

  • coordinateSpace – the orientation of the sensor. Defaults to: SIMPLE_CARTESIAN

  • id – the unique EventModel id of this component. Defaults to: DEVICE_ID_GYROSCOPE

virtual int setPeriod(int period)#

Attempts to set the sample rate of the gyroscope to the specified value (in ms).

Note

The requested rate may not be possible on the hardware. In this case, the nearest lower rate is chosen.

Note

This method should be overriden (if supported) by specific gyroscope device drivers.

Parameters

period – the requested time between samples, in milliseconds.

Returns

DEVICE_OK on success, DEVICE_I2C_ERROR is the request fails.

virtual int getPeriod()#

Reads the currently configured sample rate of the gyroscope.

Returns

The time between samples, in milliseconds.

virtual int setRange(int range)#

Attempts to set the sample range of the gyroscope to the specified value (in dps).

Note

The requested range may not be possible on the hardware. In this case, the nearest lower range is chosen.

Note

This method should be overriden (if supported) by specific gyroscope device drivers.

Parameters

range – The requested sample range of samples, in dps.

Returns

DEVICE_OK on success, DEVICE_I2C_ERROR is the request fails.

virtual int getRange()#

Reads the currently configured sample range of the gyroscope.

Returns

The sample range, in g.

virtual int configure()#

Configures the gyroscope for dps range and sample rate defined in this object. The nearest values are chosen to those defined that are supported by the hardware. The instance variables are then updated to reflect reality.

Note

This method should be overidden by the hardware driver to implement the requested changes in hardware.

Returns

DEVICE_OK on success, DEVICE_I2C_ERROR if the gyroscope could not be configured.

virtual int requestUpdate()#

Poll to see if new data is available from the hardware. If so, update it. n.b. it is not necessary to explicitly call this function to update data (it normally happens in the background when the scheduler is idle), but a check is performed if the user explicitly requests up to date data.

Note

This method should be overidden by the hardware driver to implement the requested changes in hardware.

Returns

DEVICE_OK on success, DEVICE_I2C_ERROR if the update fails.

virtual int update(Sample3D s)#

Stores data from the gyroscope sensor in our buffer, and perform gesture tracking.

On first use, this member function will attempt to add this component to the list of fiber components in order to constantly update the values stored by this object.

This lazy instantiation means that we do not obtain the overhead from non-chalantly adding this component to fiber components.

Returns

DEVICE_OK on success, DEVICE_I2C_ERROR if the read request fails.
