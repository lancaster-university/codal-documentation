class PinPeripheral#

Subclassed by codal::Button, codal::I2C, codal::PulseIn, codal::SPI, codal::Serial, codal::SingleWireSerial

Public Functions

virtual int releasePin(Pin &pin)#

Method to release the given pin from a peripheral, if already bound. Device drivers should override this method to disconnect themselves from the give pin to allow it to be used by a different peripheral.

Parameters

pin – the Pin to be released

bool isPinLocked()#

Determines if this peripheral has locked any attached pins to this peripheral. During a locked period, any attempts to release or reassign those pins to a differnet peripheral are ignored. This mechanism is primarily useful to use functions such as Pin::setDigitalValue() within a peripheral driver, but without releasing the pin’s binding to that peripheral.

Returns

true if this peripherals pin bindings are locked, false otherwise.

void setPinLock(bool locked)#

Controls if this peripheral has locked any attached pins to this peripheral. During a locked period, any attempts to release or reassign those pins to a differnet peripheral are ignored. This mechanism is primarily useful to use functions such as Pin::setDigitalValue() within a peripheral driver, but without releasing the pin’s binding to that peripheral.

Parameters

true – if this peripherals pin bindings are to be locked, false otherwise.

int reassignPin(void *pin, Pin *newPin)#

Utility function, to assist in redirect() operations and consistent use of disconnect()/connect() by peripherals. Safely disconnects pin from any attached peripherals, upfates pin to the new pin, and attaches to the given peripheral. Also validates out NULL cases.

Parameters

  • pin – Typically a mutable instance variable, holding the current pin used by a given peripheral.

  • newPin – The pin which is replacing the value of pin.
